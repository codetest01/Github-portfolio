# 5-part problem statement template

Use this standalone when writing problem statements outside the skill.

---

## Checklist — all 5 must be answered

- [ ] **Who** — specific user type (not "users")
- [ ] **What** — exact, concrete problem (not "bad UX")
- [ ] **When / Where** — context in which it occurs
- [ ] **Why Painful** — emotional + operational frustration
- [ ] **Impact** — what the user does as a result

If any one is missing → rewrite.

---

**Statement:**
> **[User type]** face **[specific problem]** when **[context/situation]**,
> which leads to **[negative outcome or frustration].**

---

## Impact behaviour options (choose what applies)

- abandons the task entirely
- builds a manual workaround
- avoids the platform / feature
- misses deadlines or compliance requirements
- loses trust in the tool
- escalates to support
- switches to a competitor

---

## Filled example

| Part | Answer |
|---|---|
| **1. Who** | New-hire sales reps on enterprise deployments |
| **2. What** | Cannot intuitively find assigned courses, certifications, or coaching feedback — siloed across separate sections |
| **3. When / Where** | During onboarding and first 60–90 days on platform |
| **4. Why Painful** | Extra clicks and dead-end navigation make the tool feel like a burden; onboarding docs don't close the gap |
| **5. Impact** | Ramp time is extended, platform adoption stalls, managers re-explain navigation repeatedly |

**Statement:**
> **New admins and sales reps** face **fragmented, unintuitive navigation**
> when **switching between training programmes, certifications, and coaching
> rooms on Allego**, which leads to **longer ramp times, repeated support
> queries, and reluctance to self-serve the platform.**
